MAGA Project Constitution Hub

このサイトは index.html 1枚でできています。中身のデータ（カード・用語・バージョン）はすべて、ファイル内の SITE_DATA という1箇所にまとまっています。コードを書く必要はありません。generator.html でフォームを埋めて、出てきた文章をコピペするだけです。

リポジトリの構成（重要）

リポジトリのルート/
├── index.html       ← サイト本体
├── generator.html   ← カード生成ツール
├── manual.html      ← このページ
└── files/            ← 実際のWord・PDFファイルを置く場所
    ├── MAGA_Project_Constitution_v1.0_FrontMatter.docx
    ├── Core20_v1.1_Merged.docx
    └── ...

カードの「ダウンロード」ボタンは、files/ フォルダの中を見にいきます。

ファイルをこのフォルダに置かないと、ボタンを押しても404になります。新しいドキュメントを追加するときは、必ず①files/フォルダに実ファイルをアップロード、②generator.htmlの「ファイル名」欄に完全に同じファイル名を入力、の2つをセットで行ってください。

全体の流れ（3ステップ）

① generator.html を開いてフォームを埋める
        ↓
② 出てきたJSONをコピーする
        ↓
③ GitHubの index.html に貼り付けてコミットする
   （新しいファイルがあれば files/ フォルダにも一緒にアップロード）
        ↓
   Vercelが自動でデプロイ（数十秒〜数分）

① generator.html を開く

generator.html をブラウザ（PCでもスマホでも可）で開きます。上部に3つのボタンがあります。

  • カード追加 ── 新しいドキュメント（Word/PDFなど）を紹介するカードを追加したいとき
  • Glossary追加 ── 新しい用語を用語集に追加したいとき
  • Version追加 ── バージョン履歴に新しい版を追加したいとき

用途に応じてボタンを選び、フォームを埋めて「JSONを生成する」を押します。

② カードを追加する場合

入力する項目

項目説明例
セクション4つのセクションのどこに置くかField Projects
カードIDカード上部の小さい文字UR-0005 · v1.3
カード名カードのタイトル都市意思決定
説明文中身の要約PROJECT COMMONSがUR-0005の実体化候補として…
ページ数・形式参考情報8 pages · docx
タグカンマ区切りで複数入力可Field, UR-0005
ファイル名実際のファイル名UR0005_Draft.docx

GitHubでの貼り付け先

index.html の中の SITE_DATA.sections を探します。4つのセクション（foundation / core20 / living / field）があり、それぞれに cards: [ ... ] という配列があります。該当セクションの最後のカードの } の直後に , を追加し、その後にコピーしたJSONを貼り付けます。

cards: [
  { ...既存のカード1... },
  { ...既存のカード2... },     ← ここに , を足す
  { ...ここに新しいJSONを貼り付け... }   ← 追加分
]

③ Glossary（用語）を追加する場合

「用語」と「定義」の2つだけ入力します。生成されたJSONは、SITE_DATA.glossary の配列の最後に、同じ要領で貼り付けます。

④ Version History（更新履歴）を追加する場合

新しいバージョン番号・フェーズ名・説明を入力します。

注意が1つだけあります。

現在の最新版には current: true という印がついていて、これが金色でハイライトされます。新しいバージョンを追加するときは、今ある current: true を false に書き換えてから、新しいJSONを追加してください。ページ上部の hero-version と SITE_DATA.version の文字列も手動で書き換える必要があります。

よくある失敗

  • カンマの付け忘れ ── JSONを配列に追加するとき、前の要素の後に,を忘れると、サイト全体が真っ白になります。
  • 引用符の種類 ── 日本語入力中に全角の記号が混ざるとJSONが壊れます。generator.htmlの出力をそのまま使えば安全です。
  • ファイル名の不一致 ── カードに書いたファイル名と、files/フォルダの実際のファイル名が1文字でも違うとリンク切れになります。

動作確認の仕方

貼り付けてコミットしたら、Vercelのデプロイ完了を待ってからブラウザでサイトを開き、該当セクションを開いて新しいカードが表示されているか確認してください。もし白い画面になった場合は、直前のコミットを取り消す（GitHubの履歴からrevertする）のが一番早い復旧方法です。
