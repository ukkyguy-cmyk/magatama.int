# MAGA Project Constitution Hub — 更新マニュアル

このサイトは `index.html` 1枚でできています。
中身のデータ（カード・用語・バージョン）はすべて、ファイル内の `SITE_DATA` という1箇所にまとまっています。
コードを書く必要はありません。**generator.html でフォームを埋めて、出てきた文章をコピペするだけ**です。

---

## リポジトリの構成（重要）

```
リポジトリのルート/
├── index.html       ← サイト本体
├── generator.html   ← カード生成ツール（公開不要、手元用）
├── MANUAL.md         ← このファイル
└── files/            ← 実際のWord・PDFファイルを置く場所
    ├── MAGA_Project_Constitution_v1.0_FrontMatter.docx
    ├── Core20_v1.1_Merged.docx
    └── ...
```

**カードの「ダウンロード」ボタンは、`files/` フォルダの中を見にいきます。** ファイルをこのフォルダに置かないと、ボタンを押しても404になります。新しいドキュメントを追加するときは、必ず

1. `files/` フォルダに実ファイルをアップロード
2. generator.htmlの「ファイル名」欄に、`files/` フォルダに置いたのと**完全に同じファイル名**を入力

の2つをセットで行ってください。ファイル名が1文字でも違うとリンク切れになります。

---

## 全体の流れ（3ステップ）

```
① generator.html を開いてフォームを埋める
        ↓
② 出てきたJSONをコピーする
        ↓
③ GitHubの index.html に貼り付けてコミットする
   （新しいファイルがあれば files/ フォルダにも一緒にアップロード）
        ↓
   Vercelが自動でデプロイ（数十秒〜数分）
```

---

## ① generator.html を開く

`generator.html` をブラウザ（PCでもスマホでも可）で開きます。
上部に3つのボタンがあります。

- **カード追加** ── 新しいドキュメント（Word/PDFなど）を紹介するカードを追加したいとき
- **Glossary追加** ── 新しい用語を用語集に追加したいとき
- **Version追加** ── バージョン履歴に新しい版を追加したいとき

用途に応じてボタンを選び、フォームを埋めて「JSONを生成する」を押します。

---

## ② カードを追加する場合

### 入力する項目

| 項目 | 説明 | 例 |
|---|---|---|
| セクション | 4つのセクションのどこに置くか | Field Projects |
| カードID（ラベル） | カード上部の小さい文字 | UR-0005 · v1.3 |
| カード名 | カードのタイトル | 都市意思決定 |
| 説明文 | 中身の要約 | PROJECT COMMONSがUR-0005の実体化候補として… |
| ページ数・形式 | 参考情報 | 8 pages · docx |
| タグ | カンマ区切りで複数入力可 | Field, UR-0005 |
| ファイル名 | 実際のファイル名（あれば） | UR0005_Draft.docx |

「JSONを生成する」を押すと、下に生成結果と次の手順が表示されます。

### GitHubでの貼り付け先

`index.html` の中の `SITE_DATA.sections` を探します。4つのセクション（foundation / core20 / living / field）があり、それぞれに `cards: [ ... ]` という配列があります。

該当セクションの `cards` 配列の**最後のカードの `}` の直後に `,` を追加し、その後にコピーしたJSONを貼り付けます。**

```js
cards: [
  { ...既存のカード1... },
  { ...既存のカード2... },     ← ここに , を足す
  { ...ここに新しいJSONを貼り付け... }   ← 追加分
]
```

## ③ Glossary（用語）を追加する場合

「用語」と「定義」の2つだけ入力します。生成されたJSONは、`SITE_DATA.glossary` の配列の最後に、同じ要領で `,` を足してから貼り付けます。

---

## ④ Version History（更新履歴）を追加する場合

新しいバージョン番号・フェーズ名・説明を入力します。

**注意が1つだけあります。** 現在の最新版には `current: true` という印がついていて、これが金色でハイライトされる仕組みです。新しいバージョンを追加するときは、

1. 今ある `current: true` を `current: false` に書き換える（または `current: true,` の行ごと削除する)
2. 新しく生成したJSON（`current: true` 付き）を配列の最後に追加する
3. ページ上部の `hero-version` の文字列と、`SITE_DATA.version` の文字列も、新しいバージョン名に書き換える

この3つ目だけは自動化されていないので、`index.html` 内を検索（Ctrl+F / Cmd+F）して `version: "Version 1.3 · 2026"` の部分を手動で書き換えてください。

## よくある失敗

- **カンマの付け忘れ** ── JSONを配列に追加するとき、前の要素の後に `,` を忘れると、サイト全体が真っ白になります。貼り付けたら必ずカンマの有無を確認してください。
- **引用符の種類** ── 日本語入力中に全角の `"` や `,` が混ざるとJSONが壊れます。generator.htmlが出力したものをそのまま使えばこの問題は起きません。手で文章を触った場合だけ注意してください。
- **ファイルの実体がない** ── カードの「ファイル名」欄に書いた名前と、実際にリポジトリに置いてあるファイル名が違うと、リンクは表示されても開けません（このサイトはリンクを自動生成しないため、実害はありませんが、記録として正確にしておくと後で混乱しません）。

---

## 動作確認の仕方

貼り付けてコミットしたら、Vercelのデプロイ完了を待ってからブラウザでサイトを開き、該当セクションを開いて新しいカードが表示されているか確認してください。もし白い画面になった場合は、直前のコミットを取り消す（GitHubの履歴から revert する）のが一番早い復旧方法です。
